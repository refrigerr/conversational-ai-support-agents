3. **Making Your First Request**:
   - To verify that your API key is working correctly, make a GET request to the health endpoint:
     ```
     GET https://api.example.com/v1/health
     ```
   - A successful response will return HTTP 200 with the body:
     ```json
     {"status": "ok", "version": "1.4.2"}
     ```
   - If you receive a 401 response, it means your API key is invalid or missing. If you receive a 503 response, it indicates the API is temporarily unavailable.

These steps will help you start using the API effectively.
